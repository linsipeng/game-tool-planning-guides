# Minecraft Enchantment Planning Terms

- **Prior work:** Number of earlier anvil operations carried by an item.
- **Prior-work penalty:** Increasing cost caused by previous operations.
- **Left input:** Item occupying the first anvil slot.
- **Right input:** Sacrificed item or book in the second slot.
- **Compatible enchantments:** Enchantments legally allowed on the same item.
- **Pairing tree:** Order used to combine multiple books before the final item.
- **Operation cost:** Experience levels required for one anvil action.
- **Total cost:** Sum of levels across the planned sequence.
- **Legal limit:** Survival restriction that may prevent an overly expensive operation.
- **Renaming cost:** Additional cost introduced when changing an item name.

Preserve each item’s real history when modelling it in [Minecraft Enchant Order](https://enchantorder.com).
