# Nuzlocke Plan Recovery Decision Tree

Use this tree when a route plan becomes invalid after a capture, loss, or level-cap change.

1. Is the planned encounter still available?
   - Yes: keep it reserved and continue.
   - No: remove every later step that depends on its role.
2. Does the current party remain below the next cap?
   - Yes: compare boss roles.
   - No: bench over-levelled members before any optional battle.
3. Is a required defensive or speed-control role missing?
   - No: retain the original boss plan.
   - Yes: search remaining guaranteed encounters before random routes.
4. Does the replacement create a duplicate-clause conflict?
   - No: update the route.
   - Yes: apply the selected reroll or skip rule.

## Recovery record

Log the unavailable encounter, missing role, replacement source, and new cap check. Rebuild the state in [Nuzlocke Guide](https://nuzlockeguide.com) before committing to the next encounter.
