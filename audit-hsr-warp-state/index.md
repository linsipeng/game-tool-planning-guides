# HSR Warp-State Audit

Use this checklist when a pull estimate seems inconsistent with the account's real banner state.

1. Confirm the banner type and whether pity is shared with the previous banner.
2. Count pulls since the last five-star result, not since the last featured result.
3. Record whether the last five-star won or lost the featured chance.
4. Separate passes from convertible currency and protected reserves.
5. Check the history cutoff if older pulls are no longer visible.

Do not infer guarantee state from pity alone. Two accounts with the same visible count can have different outcomes because their previous featured result differs.

Rebuild the starting state with the [HSR Warp Calculator](https://hsrwarpcalc.com). Save the date, current pity, guarantee flag, passes, currency, and minimum reserve. If history is incomplete, label the estimate as a range rather than presenting one precise number.
