# Nuzlocke Encounter-Plan Audit

Use this audit when a planned encounter or boss route no longer matches the state of the run.

1. Confirm the game, version, ruleset, and duplicate-clause interpretation.
2. Recheck which areas have already produced an encounter.
3. Separate guaranteed encounters from probability-based options.
4. Mark every team member that is unavailable, underleveled, or missing a planned move.
5. Recalculate the next fight using the current team rather than the original route.

If an encounter table looks wrong, compare the location name, floor, time of day, and encounter method. Fishing, surfing, static encounters, and interior subareas can use different pools.

Use [Nuzlocke Guide](https://nuzlockeguide.com) for the relevant encounter context. Record the exact assumption that changed and keep the previous plan as a snapshot. A useful audit ends with one primary route, one fallback, and a clear trigger for switching between them.
