# Sky Event Schedule FAQ

## Why does the event time look different from a guide?

The guide may use another timezone or an older seasonal schedule. Confirm the current local countdown.

## How much travel buffer should I use?

Use measured travel time plus loading and connection margin; five minutes is a practical default when the window allows it.

## Is an event still worth attempting after it starts?

Only if arrival leaves enough time to satisfy participation and claim requirements.

## What should be removed from a late route first?

Remove optional detours before reset-sensitive claims or the return step.

## Why is an exact-fit plan risky?

It assumes zero delay from loading, navigation, server response, or mistakes.

## When should I refresh the schedule?

After timezone travel, daylight-saving changes, seasonal updates, or device-clock corrections.
