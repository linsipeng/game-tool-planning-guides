# Sky Schedule QA Checklist

Use this checklist before relying on a reset-sensitive route.

## Clock

- [ ] Local timezone is correct.
- [ ] Device time and displayed time agree.
- [ ] Daily reset is distinguished from event start.
- [ ] Daylight-saving or travel changes are considered.

## Route

- [ ] Travel time is included.
- [ ] Event duration is not confused with its start interval.
- [ ] A five-minute safety buffer is reserved where possible.
- [ ] Optional activities are clearly marked.

## Recovery

- [ ] Late-arrival cutoff is defined.
- [ ] Reset-sensitive claims happen before optional detours.
- [ ] A fallback route exists.
- [ ] Exact-fit schedules are labelled risky.

Confirm the live times in [Sky Clock Online](https://skyclockonline.com) immediately before starting.
