# Planning Three HSR Banner Outcomes Before Pulling

One expected-value number does not describe the decisions inside a banner session. A practical plan should cover a good outcome, a normal outcome, and a costly outcome before any pulls are spent.

First record current pity, guarantee state, available passes, and the currency that must remain untouched. Define the good case as an early featured result, the normal case as reaching a typical decision point, and the costly case as losing the featured chance and approaching the next guarantee.

For every case, write the action that follows. You might stop immediately, continue to a fixed ceiling, or preserve the guarantee for a future banner. The action matters more than guessing which case will occur.

Model the starting values with the [HSR Warp Calculator](https://hsrwarpcalc.com), then compare every scenario with the same minimum reserve. If the costly case would break an important future plan, lower the current ceiling before opening the banner.
