# Sky Schedule Misconceptions

## “Event start and daily reset are the same clock”

They are separate boundaries. An activity may repeat on an interval while rewards reset at another time.

## “If the event has started, arriving anytime is useful”

Travel and participation requirements can create a late-arrival cutoff.

## “An exact fit is a safe schedule”

Exact fits have no resilience for loading, connection, or navigation delay.

## “The displayed time is universal”

The same event can be shown differently after timezone, device-clock, or daylight-saving changes.

## “Optional activities can be removed at the end”

Reset-sensitive claims should be protected first; optional detours are removed before the return step.
