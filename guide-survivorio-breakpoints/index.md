# Finding Useful Upgrade Breakpoints in Survivor.io

Not every numerical upgrade changes a real run. The useful question is whether an upgrade crosses a breakpoint: fewer attacks to defeat an enemy, surviving an extra hit, or completing a wave before pressure increases.

Create a baseline with current equipment, level, and relevant bonuses. Then test one planned upgrade at a time. Record the percentage change, but also translate it into an encounter result such as time to defeat, attacks required, or remaining health.

Small gains can matter when they cross a threshold, while a larger headline gain may not change the current bottleneck. Consider resource cost and the next alternative upgrade, especially when materials are shared between slots.

Use the [Survivor.io Calculator](https://survivoriocalc.com) to save consistent scenarios. Rank upgrades by the practical breakpoint reached per unit of scarce resource, not only by the largest displayed damage number.
